# Decisions — ferry

Append-only log of session decisions. One dated line each. Promote a decision to
a `docs/decisions/` ADR when it shapes architecture or is expensive to reverse.

- 2026-07-05: GitHub Actions must stay within the free tier — no reliance on paid minutes. Platform coverage moves to committed .githooks/pre-push preflight gates (run everywhere via the global core.hooksPath dispatcher); private-repo CI runs ubuntu-latest by default, switchable with the CI_RUNNER repo variable if development ever moves off macOS. Rejected: nektos/act (Docker dep, Linux-only emulation); making ragd public (not desired).
- 2026-07-05: W4 state-file migration backs up via a write-once sibling copy (`statefile.BackupForMigration`, `.pre-v1.bak`), NOT the existing Backuper the plan text names — routing migrations through the Backuper would inject ferry's own bookkeeping into the restore baseline, which restore would then try to revert. Confirmed by independent review.
- 2026-07-05: W1 keeps the install.sh checksum commit-back (W3 removes it) but narrowed to a one-file commit on the fetched default-branch tip via a detached worktree — the tag's tree must never reach the branch. Rejected: amending W1's done-criterion to accept non-fast-forward failure for older-commit tags.
- 2026-07-05: W1's live done-criteria demos (older-commit tag; red-suite tag → no release) deferred to a disposable fork before W7 — this repo's plan forbids tag deletion, so throwaway tags may not be created here.
- 2026-07-05: bin/ferry-* binaries untracked (post-v0.4.0): installation is release-only and W3's security review called committed binaries vestigial; they are build artifacts regenerated by `make build` and shipped as attested release assets. Rejected: keeping them tracked for a no-network install path (that path no longer exists).
- 2026-07-06: W2 agents capture-back captures only the CLEANLY-reversible agents targets — asset files (1:1 copy; shared + local routes) and single-source instruction targets (general.md / coding.md; shared route only). SourceCombined targets (codex/gemini AGENTS.md) are REFUSED (header+general+coding cannot be decomposed — the plan's "never guess a winner"). True divergence = the deployed file AND the derived repo source both moved past the last-deployed baseline (dotfile.StateConflict); refused with a diff, never auto-merged. Adopt-new is registry-driven: scan each resolved asset mapping's $HOME target dir for undeployed regular files. Local route wired for assets via local/agents/<source>/<rel> (Plan reads it, local wins), mirroring dotfiles; instruction targets are shared-only so a per-target local capture can't silently change every harness that shares general/coding.
- 2026-07-06: W3 config-file terminals (Alacritty/kitty/WezTerm) are a NEW domain gated by `[manage] terminals = true` (plural — `terminal` singular is the existing macOS Apple-Terminal defaults domain), carried like dotfiles via a built-in registry in internal/termcfg mirroring the agents registries. The generic built-ins+overlay+selection resolver was EXTRACTED from internal/agents into internal/registry (behaviour-preserving) so all three registries share one copy rather than a third. Each terminal maps a repo source under terminals/ (a directory or a single file) to a $HOME target; a directory fans out file-by-file, a single file (wezterm.lua) is one item. The .local overlay is local/terminals/<source>/<relpath> (per-file, local wins — the per-machine colour-scheme case). Deploy routes through dotfile.ApplyContentDeferred (same three-way classify/conflict/backup as dotfiles+agents), so terminal items are a guided kind (own "terminals" risk group; risky overwrite fails closed). Terminals are NOT secret-rendered (like agents, unlike dotfiles) — a design choice for W3. DEFERRED: scoped `ferry restore terminals` and a cumulative target record; full `ferry restore` already reverts terminal files via the engine's immutable baseline, and the de-scope warning points there. GNOME Terminal/dconf deferred per plan (needs a dump/load bridge, not a file copy).
2026-07-06 v0.6.0 scope: security (fn-4+restore lock) already shipped as v0.5.3; v0.6.0 = IA adoption (§§1-9) + fn-6.1 leaf-swap TOCTOU. Rejected: renumbering per plan R1 (moot).
2026-07-06 Record placement (A1): DROP the abcd 7-folder Record; keep docs/{decisions,plans,research} only. No development/ tree, no scaffoldDocsDirs extension. Rejected: docs/ baseline w/ 4 new folders, top-level development/.
